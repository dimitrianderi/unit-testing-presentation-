Disadvantages of unit testing

Writing tests can take longer than the code itself.

Unit testing does not guarantee that all bugs will be found.

Unit testing cannot reveal systemic bugs in the product as a whole.

Unit test structure

One unit-test
					
						test('description', () => {
							expect(data).toBe(value);
						});

Group unit-tests
					
						describe('group description', () => {
							test('description', () => {
								expect(data).toBe(value);
							});

							test('description', () => {
								expect(data).toBe(value);
							});
						}

Example 1

Our function
						
							function sum(a, b) {
								return a + b;
							}

Get the amount
					
						test('adds 1 + 2 to equal 3', () => {
							expect(sum(1, 2)).toBe(3);
						});

Example 2

Our function
						
							function oldPerson(born) {
								return 2022 - born;
							}

Get a number greater than 60
					
						test('person at least 60 years old', () => {
							expect(oldPerson(1940)).toBeGreaterThan(60);
						});

Example 3

Our function
						
							function people(user) {
								user.age = 30;
								return user;
							}

Compare two objects
					
						test('objects must be equal', () => {
							expect(people({name: 'Alex'})).toEqual({name: 'Alex', age: 30});
						});
